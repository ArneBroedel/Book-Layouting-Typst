# Plan: Skill-Wiederverwendungs-Pilot

Siehe [spec.md](spec.md). Status: **geplant / nicht begonnen.**
Lifecycle: gefundene Skill-Lücken sind der Haupt-Harvest (→ `typst-writer`/`typst-extension`); danach
Track nach `_archive/` (siehe `docs/KNOWLEDGE-MAP.md`).

## Phasen

- [ ] **Phase 0 — Thema & Form wählen.** Eine bewusst andere Domäne festlegen (z. B. technisches
  Datenblatt oder wissenschaftliche Kurzmitteilung), Umfang ~6–8 Seiten, Zielort `pilots/<name>/`.
- [ ] **Phase 1 — Skill-geführt bauen.** NUR über Skills/Guides + vorhandene `components/`/`theme.typ`
  arbeiten; eigene Token-Palette als Beweis der Konfigurierbarkeit. Jede genutzte Skill-Regel notieren.
- [ ] **Phase 2 — Reuse-Report.** Protokoll: getriggerte Skills, 1:1-Wiederverwendung vs. Improvisation,
  konkrete Skill-Lücken, grober Aufwand (Zeit/Iterationen).
- [ ] **Phase 3 — Rückfluss.** Mindestens eine gefundene Lücke als Skill-Verbesserung umsetzen
  (Harvest), Report finalisieren.

## Abnahme
- Pilot-Artefakt baut reproduzierbar; Reuse-Report + mindestens eine Skill-Verbesserung (oder begründetes „keine Lücken").
