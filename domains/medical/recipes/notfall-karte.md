# Genre recipe: notfall-karte (Domain medical)

**Produkt B** — not part of form-catalog core.  
**Functions** the presentation must serve (Media uses as checklist; Tech maps to forms).

## Functional requirements

1. **Gatekeeper / danger first** — reader sees stop/act cue within first screenful (cave or memo).  
2. **Recognition triggers** — scannable list or protocol, not a wall of prose.  
3. **Action / triage path** — clear urgency → action (table *or* protocol; Media chooses ideal).  
4. **Immediate measures** — ordered steps preferred over dense narrative.  
5. **Confusion / DDx (short)** — optional ddx or comparison form.  
6. **Cross-links** — compact, de-emphasized vs. action content.  
7. **Sources** — compact; de-emphasized on early pages.

## Validator minima (suggested for compose genre config)

- distinct catalog forms ≥ 4  
- first page / early content includes `cave` or `memo`  
- not only `styled-table` + body  

## Recommended stable forms (starting map — not exclusive)

`chapter-opener`, `cave`, `memo`, `styled-table`, `protocol-steps` (planned), `ddx-box`, `callout`, `section-break`

## Anti-patterns

- Everything as one giant table  
- Cave on every paragraph  
- Sources before action  
- Catalog-only thinking in Media Brief phase
