# Genre recipe stub: lerntext (Domain medical)

**Status:** stub (Wave 1) — expand when pilot needs full minima.  
**Produkt B** — not form-catalog core.

## Purpose

Concept-first teaching text: objectives → definitions → explanation → summary.  
Not action-under-time-pressure (see `notfall-karte`).

## Functional checklist (draft)

1. **Learning goal** visible early (`learning-objectives` or equivalent intent).  
2. **Key terms** boxed or highlighted (`definition`, `memo`).  
3. **Explanatory body** with scannable structure (section breaks, callouts).  
4. **Takeaway / summary** before deep hintergrund (`summary-box`, `memo`).  
5. **Sources** de-emphasized trailing.

## Validator minima (placeholder — no YAML yet)

- distinct_forms_min: **3** (suggested)  
- early: learning-objectives **or** definition/memo  
- forbid: pure prose + single table only (soft)

## Recommended forms (non-exclusive)

`chapter-opener`, `learning-objectives`, `definition`, `memo`, `callout`, `summary-box`, `section-break`, `styled-table` (sparingly)

## Anti-patterns (draft)

- Emergency-card layout for pure concept chapters  
- Definition spam without narrative thread  
- No summary after long exposition
