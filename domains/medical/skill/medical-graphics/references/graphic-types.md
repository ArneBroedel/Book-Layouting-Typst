# Graphic types (didactic content graphics)

**Not anatomy-first.** Anatomy/spatial is one type among many.

| Type | Learner job | Typical ideal | Code tendency | Asset tendency |
|---|---|---|---|---|
| **decision-flow** | choose / act | Branching, urgency lanes | high (Fletcher/CeTZ/table) | medium |
| **procedure-sequence** | execute | Weighted steps | high | low |
| **mechanism-schema** | understand (dual coding) | Segmented schema + text | medium | **high** if organic |
| **comparison-matrix** | distinguish | 2-col / matrix | high | low |
| **cue-cluster** | recognize | Icon+label tiles | medium | high for custom icons |
| **relation-map** | relate | Nodes/edges, concepts | medium–high | medium |
| **timeline-course** | course over time | Timeline | high | low |
| **priority-landscape** | see urgency | Lanes, stacks, weighted bands | high | low |
| **spatial / anatomy** | localize | Markers, contours | hybrid | **high** (base SVG) |
| **data-chart** | quantify | Bar, KM, forest, ROC… | **very high** (CeTZ) | rare |

## Mapping from Media hypotheses

Media writes prose pattern names (e.g. “branching flowchart”, “dual-coding mechanism”). Graphics maps to a **type** + **mode** without shrinking the ideal.

## Anti-patterns

- Treating every figure as anatomy plate  
- Equal visual weight for decoration and action path  
- Encoding Critical Claims only inside non-text raster (unreadable / unsearchable / hard to revise)  
- A/B of unrelated content (must same unit + same claims)  
- Dropping wishlist graphics without Decision Note attempt  

## Dual coding reminder

For **mechanism-schema**: complementary channels (visual structure + concise text), labels near referents. Do not invent clinical simplification — that is Content (C).
