# Checklist: Developmental edit (core · Q2)

Use during Content-Review. Mark findings in the Review Report (`struktur`, `didaktik`, `konsistenz`, …).

## Structure

- [ ] Audience and chapter goal are clear (or flagged if missing)
- [ ] Section order supports progressive understanding
- [ ] Headings hierarchy is coherent (no orphan H3s / random jumps)
- [ ] Redundant sections identified
- [ ] Internal contradictions flagged

## Cognitive load

- [ ] Sections are scannable (not walls of text without anchors)
- [ ] One main idea per major block where reasonable
- [ ] Definitions appear before heavy use of terms
- [ ] Takeaway / action summary present if genre expects it

## Learning path (generic — not medical-profile deep)

- [ ] Declared goals (if any) map to body coverage
- [ ] “What should the reader *do* or *decide*?” is answerable — else `should`/`block`

## Didactic reduction & emphasis (content-side)

Lean checks only — **flag**, do not redesign layout (that is Media). Do not invent missing clinic.

- [ ] If the lead/summary **simplifies**, critical conditions / exceptions still appear **in the body near the action** (not only in a distant appendix)
- [ ] Action or decision instructions do not depend on a “full story later” that never arrives
- [ ] Background/excursus is clearly secondary to the reader’s job (or flagged if it steals the spine)
- [ ] Same concept not renamed mid-chapter without reason (`konsistenz` / clarity)

Tag: usually `didaktik` or `struktur`. Layout loudness is **out of scope** here.

## Severity guidance

| Issue | Typical severity |
|---|---|
| Missing spine / unusable structure | `block` |
| Lead simplification that strands safety conditions | `block` / `should` |
| Major redundancy or confusion | `should` |
| Wording polish only | `nit` |
