# Checklist: Critical Claims audit (core)

**Do not invent clinical facts.** Only assess what is written and what is missing as *questions*.

## Presence

- [ ] A Critical Claims section exists (or gap is a `block` / `should` per work policy)
- [ ] Claims are listed as discrete, checkable statements (not marketing prose)

## Quality per claim

- [ ] Each claim is testable (true/false or observable)
- [ ] No soft claims (“often important…”) without sharpening recommendation
- [ ] Safety-relevant body statements appear as claims when work policy requires

## Consistency

- [ ] Claims do not contradict the body
- [ ] Body does not smuggle absolute clinical rules that are absent from claims (flag for author)

## Output

Record gaps in Review Report → **Claims audit** table. Severity:

| Issue | Severity |
|---|---|
| No claims section where required | `block` or `should` (work policy) |
| Untestable / soft claim | `should` |
| Claim–body contradiction | `block` |
